# Release Manager

> 파이프라인 규칙, 산출물 체계, 역할, 변경 처리는 `SKILL.md`를 따른다.

- 머지된 PR 목록을 입력으로 릴리스를 준비한다. 산출물 ID: `RELEASE-xxxx-slug`.
- 모든 자동 게이트(CI)를 통과하고, 릴리스 승인(사람)을 받아야 배포한다.
- 배포는 단계적(Canary/Blue-Green)으로 진행하며, 이상 시 즉시 롤백한다.

## 실행 절차

### 1단계: 입력 확인

1. 릴리스에 포함할 PR 목록을 확인한다.
2. 모든 PR이 머지되었는지 확인한다.
3. 각 PR의 참조 TASK가 완료 상태(WBS 전체 DONE + 완료 기준 충족)인지 확인한다.
4. 모든 자동 게이트(CI)를 통과했는지 확인한다.

### 2단계: RELEASE 문서 생성

아래 **RELEASE 문서 템플릿**에 따라 `docs/ai-devops/RELEASE/RELEASE-xxxx-slug.md`를 생성한다.

### 3단계: 배포 전 점검

- [ ] 롤백 계획이 문서화되어 있는가?
- [ ] 관측성(대시보드/알람)이 준비되어 있는가?
- [ ] 데이터 마이그레이션이 필요하면 마이그레이션 스크립트가 검증되었는가?
- [ ] 마이그레이션 롤백 스크립트가 준비되어 있는가?
- [ ] 외부 의존 서비스(API/MQ)의 호환성이 확인되었는가?
- [ ] 릴리스 승인(사람)을 받았는가?

### 4단계: 단계 배포

배포 전략에 따라 단계적으로 배포한다. 각 단계에서 핵심 메트릭을 모니터링한다.

#### 배포 단계

1. **데이터 마이그레이션** (필요 시): 스키마/데이터 변경을 먼저 적용한다. 롤백 가능한 방식(expand-contract)을 사용한다.
2. **Canary 배포**: 전체 트래픽의 일부(5~10%)를 새 버전으로 라우팅한다.
3. **모니터링**: 오류율, 지연, 핵심 비즈니스 메트릭을 관측한다. 최소 관측 시간(예: 15분~1시간)을 확보한다.
4. **확대 또는 롤백**: 이상 없으면 트래픽 비율을 점진적으로 확대한다. 이상 발견 시 즉시 롤백한다.
5. **전체 배포**: 100% 트래픽을 새 버전으로 전환한다.

### 5단계: 배포 후 검증

- [ ] 오류율이 배포 전 대비 정상 범위인가?
- [ ] 지연(P50/P99)이 SLO 목표 이내인가?
- [ ] 핵심 비즈니스 메트릭(주문 수, 가입 수 등)에 이상이 없는가?
- [ ] 로그에 비정상 에러 패턴이 없는가?
- [ ] 알람이 발생하지 않았는가?

### 6단계: 릴리스 확정 또는 롤백

#### 확정 조건

- 배포 후 관측 기간(예: 24시간) 동안 SLO 위반 없음
- Critical/High 이슈 없음

#### 롤백 의사결정 기준

| 기준 | 롤백 |
|------|------|
| 오류율 배포 전 대비 2배 이상 증가 | 즉시 롤백 |
| P99 지연 SLO 초과 | 원인 분석 후 15분 내 미해결 시 롤백 |
| 데이터 정합성 문제 발견 | 즉시 롤백 + 데이터 복구 계획 수립 |
| 핵심 비즈니스 메트릭 10% 이상 하락 | 즉시 롤백 |
| 보안 취약점 발견 | 즉시 롤백 |

#### 롤백 실행 절차

1. 이전 버전으로 트래픽을 전환한다.
2. 데이터 마이그레이션이 있었으면 롤백 스크립트를 실행한다.
3. 롤백 후 메트릭이 정상화되었는지 확인한다.
4. 롤백 원인을 분석하고, `CHANGE_REQUEST`로 수정 계획을 수립한다.

### 7단계: 사후 정리

1. **릴리스 노트 발행**: 주요 변경 사항, 계약 변경, 알려진 이슈를 포함하여 릴리스 노트를 작성한다.
2. **운영 문서 갱신**: 대시보드/알람/런북 등 운영 관련 문서를 업데이트한다.
3. **RELEASE 문서 상태 업데이트**: 배포 완료 시 `RELEASED`, 롤백 시 `ROLLED_BACK`으로 갱신한다.
4. **TASK 상태 확정**: 모든 관련 TASK를 `DONE`으로 업데이트한다.

## 핵심 원칙

- **단계 배포 필수**: 프로덕션 배포는 반드시 Canary 또는 Blue-Green을 거친다. 전체 트래픽에 직접 배포하지 않는다.
- **롤백 계획 없는 배포 금지**: 모든 릴리스는 롤백 계획을 문서화한 후 배포한다.
- **관측 없는 확정 금지**: 배포 후 관측 기간을 거치지 않고 릴리스를 확정하지 않는다.
- **산출물 추적**: RELEASE 문서는 포함 PR/TASK를 모두 참조하여 추적성을 유지한다.
